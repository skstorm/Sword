## 트러블슈팅

### 1. Flutter 미설치
- **현상**: `flutter` 명령어를 찾을 수 없음
- **원인**: 개발 머신에 Flutter SDK가 설치되어 있지 않았음
- **해결**: `C:\dev\flutter`에 Flutter 3.29.3 SDK 다운로드 및 압축 해제
- **대책**: Windows PATH에 `C:\dev\flutter\bin` 영구 등록 (`setx PATH`)

### 2. app_flutter에 game_core 의존성 누락
- **현상**: Phase 0 완료 보고 후 검토에서 발견
- **원인**: `flutter create`로 앱 생성 후 `pubspec.yaml`에 game_core path 의존성 추가를 빠뜨림
- **해결**: `pubspec.yaml`에 `game_core: path: ../packages/game_core` 추가 + assets 등록
- **대책**: Phase 완료 시 태스크리스트 대조 검증 프로세스 도입

### 3. enhance_command.dart 보호 부적 후 리셋 버그
- **현상**: 보호의 부적이 파괴를 막았는데도 나무검으로 리셋됨
- **원인**: `handleFail()` 실행 후 `hasActiveProtection`이 `false`로 변경된 상태에서 리셋 조건 `!currentState.hasActiveProtection`을 검사하여 항상 true가 됨
- **해결**: 리셋 조건을 이벤트의 `destroyed` 플래그 기반으로 변경: `if (failEvent.destroyed && !currentState.pendingAdProtection)`
- **대책**: 상태 변경 후 이전 상태 기반 판단이 필요한 경우, 이벤트 페이로드의 플래그를 사용

### 4. enhance_logic.dart 보호 시 파편 지급 버그
- **현상**: 보호 부적으로 파괴가 방지되었는데 파편이 지급됨
- **원인**: handleFail에서 보호 여부와 무관하게 fragments를 playerData에 추가
- **해결**: 보호 시 파편 미지급 (`destroyed ? fragmentsGained : 0`), 실제 파괴 시에만 파편 지급
- **대책**: 구현계획서의 예시 코드와 실제 구현을 대조 검증

### 5. enhance_view.dart 확률 표시 버그 (2건)
- **현상 1**: +14에서 "???"로 표시됨 (스펙: +14이하 숫자)
- **원인**: 조건이 `level < 14`였음 (14 미포함)
- **해결**: `level < 15`로 수정
- **현상 2**: 현재 검의 확률이 표시됨 (다음 레벨 검의 확률이어야 함)
- **원인**: `context.state.currentSword.successRate`를 읽고 있었음
- **해결**: `swordTable.getSword(level + 1)?.successRate`로 다음 레벨 검 조회
- **대책**: 뷰 레이어 작업 시 "무엇을 표시하는가"의 데이터 소스를 구현계획서와 대조

### 6. Visual Studio Build Tools 미설치
- **현상**: `flutter run -d windows` 시 "Unable to find suitable Visual Studio toolchain"
- **원인**: Windows 데스크톱 빌드에 필요한 Visual Studio C++ Build Tools 미설치
- **해결**: `flutter run -d chrome`으로 웹 브라우저에서 실행하여 우회
- **대책**: Windows 네이티브 빌드 필요 시 Visual Studio Build Tools 설치
